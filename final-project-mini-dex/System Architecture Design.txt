# System Architecture Design

## Project Name

Mini DEX (Decentralized Exchange)

---

# 1. System Overview

The Mini DEX is a decentralized application (DApp) that enables users to swap ERC20 tokens and manage liquidity through blockchain smart contracts.

The platform consists of three major layers:

1. Frontend Layer
2. Blockchain Layer
3. Wallet Layer

---

# 3. Components

## Frontend Layer

Technology:

* React.js
* Ethers.js

Responsibilities:

* Wallet connection
* Token swap interface
* Liquidity management interface
* Display reserves and balances

---

## Wallet Layer

Technology:

* MetaMask

Responsibilities:

* User authentication
* Transaction signing
* Blockchain interaction

---

## Smart Contract Layer

Technology:

* Solidity

Contracts:

1. DevToken.sol
2. RewardToken.sol
3. MiniDEX.sol

Responsibilities:

* Store liquidity
* Execute swaps
* Manage token transfers
* Track reserves

---

# Smart Contract Logic

## Contract 1: DevToken.sol

Purpose:
ERC20 Token A

Functions:

* transfer()
* approve()
* transferFrom()
* balanceOf()

---

## Contract 2: RewardToken.sol

Purpose:
ERC20 Token B

Functions:

* transfer()
* approve()
* transferFrom()
* balanceOf()

---

## Contract 3: MiniDEX.sol

Core Functions:

### addLiquidity()

Input:

* Token A Amount
* Token B Amount

Process:

* Transfer tokens to DEX
* Increase reserves

Output:

* Liquidity added

---

### removeLiquidity()

Input:

* Liquidity Amount

Process:

* Return proportional tokens

Output:

* Liquidity removed

---

### swap()

Input:

* Token In
* Amount

Process:

* Calculate output amount
* Transfer tokens

Output:

* Tokens swapped

---

### getReserves()

Output:

* Token A Reserve
* Token B Reserve

Purpose:
Display pool status
